Next Greater Element I

A monotonic decreasing stack walks nums2 once, building a next-greater map lookup.

nums2 (Scan Array)
How It Works

The algorithm scans nums2 from left to right, maintaining elements in a monotonic decreasing stack (top of the stack is always the smallest element).

Key Concept: Monotonicity

When we see a number larger than the stack's top element, it means we found the "next greater element" for that top element. We pop it from the stack, record the mapping in our hash map, and repeat until the stack is empty or the top is larger than the current number. Then, we push the current number.

Algorithm Steps

  1. 1 Scan nums2: Iterate through nums2 from left to right.
  2. 2 Pop & Map: While stack is not empty and current element is greater than stack's top, pop the top and map popped → current.
  3. 3 Push: Push the current element onto the stack.
  4. 4 Lookup: For each element in nums1, retrieve its next greater value from the map (or -1 if not mapped).

Complexity

Time: O(n + m) — each element is pushed and popped at most once.

Space: O(n) — to store the stack and hash map mappings.
